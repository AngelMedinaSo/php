USERS TABEL - Kolommen uitleg
==============================

Kolom 1: id
  Type:   INT, AUTO_INCREMENT, PRIMARY KEY
  Waarom: Elke gebruiker heeft een uniek nummer nodig.
          Je kan niet op username vertrouwen, want twee
          mensen kunnen dezelfde naam willen. Een ID is
          altijd uniek en verandert nooit.

Kolom 2: username
  Type:   VARCHAR(100)
  Waarom: De naam waarmee de gebruiker inlogt.
          Dit is zichtbaar voor de gebruiker zelf.

Kolom 3: password
  Type:   VARCHAR(255)
  Waarom: Het wachtwoord van de gebruiker.
          Dit wordt NIET als normale tekst opgeslagen,
          maar als een hash (versleuteld).
          Als iemand de database steelt, ziet hij
          alleen een onleesbare code, niet het echte
          wachtwoord.

==============================
DENKNAVRAGEN
==============================

Waarom heeft een gebruiker een eigen ID nodig?
  Twee gebruikers kunnen dezelfde username hebben.
  Een ID is altijd uniek. De database gebruikt het ID
  intern om alles aan elkaar te koppelen.

Waarom slaan we een wachtwoord niet als normale tekst op?
  Als de database gehackt wordt, kan iemand direct
  alle wachtwoorden lezen. Met een hash (zoals bcrypt)
  is het wachtwoord onleesbaar opgeslagen. Zelfs de
  beheerder van de website kan het wachtwoord dan
  niet zien.

==============================
SQL om de tabel aan te maken:
==============================

CREATE TABLE users (
  id       INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(100) NOT NULL,
  password VARCHAR(255) NOT NULL
);
