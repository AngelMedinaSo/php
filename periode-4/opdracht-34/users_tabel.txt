USERS TABLE

id       - INT, AUTO_INCREMENT, PRIMARY KEY - unique number for each user
username - VARCHAR(100)                     - the name used to login
password - VARCHAR(255)                     - the password, stored as a hash (not plain text)

Why do we need an id?
Two users can have the same username. The id is always unique.

Why not store the password as plain text?
If the database gets hacked, the hacker can read all passwords.
A hash makes it unreadable.

SQL:
CREATE TABLE users (
  id       INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(100) NOT NULL,
  password VARCHAR(255) NOT NULL
);
